Изучите основы

React Native похож на React, но он использует нативные компоненты вместо web-компонентов для построения блоков. Для понимания основной структуры React Native приложения, вам надо понять некоторые основные концепты React, как JSK, компоненты, state и props. Если вы уже знаете React, вам всё ещё надо выучить некоторые особенности React Native, например нативные компоненты. Этот туториал нацелен на всех, вне зависимости от того, есть у вас опыт с React или нет.

Что здесь происходит?

Некоторые вещи здесь могут вам показаться не похожими на JavaScript. Не паникуйте. Это будущее!

Прежде всего, ES2015 (также известный как ES6) это набор улучшений для JavaScript который теперь часть официального стандарта, но всё ещё не поддерживаемый некоторыми браузерами, поэтому пока ещё не часто используется в web-разработке. React Native поддерживает ES2015, так что можете использовать не боясь проблем с совместимостью. Import, from, class и extends – это всё нововведения ES2015. Есил вы не знакомы с ES2015, вы, вероятно, сможете понять его просто прочитав пример кода как в этом туториале. Если хотите, то на этой странице есть хороший обзор нововведений ES2015.

Другая необычная вещь в этом примере кода – это <View><Text>Hello World!<View><Text>. Это JSX – синтаксис для встраивания XML в JavaScript. Много фреймворков используют специальный шаблонный язык, который позволяет встраивать код в язык разметки. JSX позволяет писать на языке разметки внутри кода. Это выглядит как HTML в web, только вместо таких вещей, как <div> или <span>, вы используете React компоненты. В этом случае, <Text> это встроенный компонент, который просто показывает текст, а View это как <div> или <span>.
