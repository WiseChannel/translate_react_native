Сеть

Многие мобильные приложения загружают ресурсы с удалённого URL. Возможно, вы захотите сделать POST запрос к REST API, или вам просто может понадобиться получить набор статичных данных с другого сервера.

Использование Fetch

React Native предоставляет Fetch API для ваших потребностей с сетями. Fetch покажется вам знакомым, если вы уже использовали XMLHttpRequest или другие сетевые API. Вы можете обратиться к руководству MDN по использованию Fetch для получения дополнительной информации.

Не забывайте отлавливать любые ошибки, которые могут быть сгенерированы Fetch, иначе они будут отброшены молча.

По умолчанию, iOS заблокирует любой запрос, который не зашифрован с помощью SSL. Если вам нужно извлечь из открытого текста URL-адрес (который начинается с http), вам сначала надо добавить исключение App Transport Security. Более безопасно добавлять исключения только для тех доменов, к которым вы заранее знаете, что потребуется доступ. Если домены не известны до времени выполнения, то вы можете полностью отключить ATS. Обратите внимание, что с января 2017 года проверка Apple App Store будет требовать разумного обоснования дя отключения ATS. Смотрите документацию Apple для дополнительной информации.

Использование других сетевых библиотек

XMLHttpRequest API встроен в React Native. Это значит, что вы можете использовать библиотеки от третьих лиц, например frisbee или axios, или можете спользовать XMLHttpRequest API, если вам так удобнее.

Модель безопасности для XMLHttpRequest отличается от сетевой, поскольку в нативных приложениях отсутствует концепция CORS.

Поддержка WebSocket

React Native также поддерживает WebSockets,- протокол, который обеспечивает полнодуплексные каналы связи по одному TCP-соединению.

Известные проблемы с Fetch и аутентификацией базирующейся на cookie

Представленные ниже опции не работают с fetch

  • redirect:manual
  • credentials:omit
  • Наличие одинаковых заголовков на Android приведёт к появлению только самого последнего. Временное решение можно найти здесь: https://github.com/facebook/react-native/issues/18837#issuecomment-398779994
  • Аутентификация, основанная на cookie пока что не стабильна. Вы можете посмотреть некоторые проблемы здесь: https://github.com/facebook/react-native/issues/23185
  • На iOS, при перенаправлении через 302, если отсутствует заголовок Set-Cookie, cookie не установлен должным образом. Поскольку перенаправление не может быть обработано вручную, это может вызвать сценарий, в котором возникают бесконечные запросы, если перенаправление является результатом истёкшего сеанса.
