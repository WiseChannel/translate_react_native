Больше ресурсов

Если вы только что читали статьи на этом веб-сайте, вы сможете создать довольно классное приложение React Native. Но React Native - это не просто продукт, созданный одной компанией, это сообщество тысяч разработчиков. Так что, если вы заинтересованы в React Native, вот несколько связанных вещей, которые вы можете попробовать.

Популярные библиотеки

Если вы используете React Native, вы, вероятно, уже знаете о React . Поэтому я чувствую себя немного глупо, упоминая об этом. Но если всё таки нет, посмотрите React - это лучший способ создать современный сайт.

Один общий вопрос - как обработать «state» вашего приложения React Native. Самая популярная библиотека для этого - Redux. Не бойтесь того, как часто Redux использует слово «reducer» - это довольно простая библиотека, а также есть хорошая серия видеороликов, объясняющих это.

Если вы ищете библиотеку, которая делает что-то конкретное, посмотрите Awesome React Native, список курируемых компонентов, в которых также есть статьи и другие материалы.

Примеры

Попробуйте приложения из «витрины» , чтобы увидеть, на что способен React Native! Есть также несколько примеров приложений на GitHub . Вы можете запускать приложения на симуляторе или устройстве, и вы можете видеть исходный код этих приложений.

Люди, которые создали приложение для конференции F8 в Facebook, также открыли код и написали подробную серию руководств . Это полезно, если вам нужен более подробный пример, более реалистичный, чем большинство примеров приложений.

Расширение React Native

  • Другие разработчики пишут и публикуют модули React Native для npm и открывают их на GitHub.
  • Создание модулей помогает развивать экосистему и сообщество React Native. Мы рекомендуем писать модули для ваших сценариев использования и делиться ими на npm.
  • Прочтите руководства по Native модулям (iOS, Android) и компонентам Native UI (iOS, Android), если вы заинтересованы в расширении собственных функций.
  • Ищете готовый компонент? Посмотрите JS.coach или Native Directory, чтобы узнать, что создает сообщество.

Инструменты разработки

Nuclide - это среда разработки, которую Facebook использует для разработки JavaScript. Убийственная особенность Nuclide - его способность к отладке. Он также имеет отличную встроенную поддержку потока. VS Code - еще одна IDE, популярная среди разработчиков JavaScript.

Ignite - это начальный набор, который использует Redux и несколько различных общих библиотек пользовательского интерфейса. Он имеет интерфейс командной строки для генерации приложений, компонентов и контейнеров. Если вам нравятся все отдельные технические решения, Ignite может подойти вам.

App Center - это сервис от Microsoft, который позволяет легко развертывать живые обновления в вашем приложении React Native. Если вам не нравится проходить через процесс магазина приложений для развертывания небольших настроек, а также вам не нравится настраивать собственный бэкэнд, попробуйте App Center.

Expo - это среда разработки, плюс приложение, которое позволяет вам создавать приложения React Native в среде разработки Expo, не касаясь Xcode или Android Studio. Если вы хотите, чтобы React Native был еще более популярным в JavaScripty и Webby, попробуйте Expo.

Yoga - это автономный механизм компоновки, который выходит за рамки React Native и позволяет инженерам-разработчикам быстро создавать макеты для нескольких платформ с высоко оптимизированным механизмом компоновки с открытым исходным кодом, разработанным с учетом скорости, размера и простоты использования.

Bugsnag , Microsoft App Center и Sentry предоставляют отличные сервисы мониторинга сбоев и ошибок для приложений React и React Native. Эти сервисы позволяют в режиме реального времени активно отслеживать сбои и проблемы, возникающие в ваших приложениях, чтобы вы могли быстро их исправить и улучшить взаимодействие с пользователем.

Инструменты разработчика React отлично подходят для отладки приложений React и React Native.
