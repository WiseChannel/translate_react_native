State

Существует два типа данных, которые контролируют компонент: props и state. Props устанавливаются родителями и они зафиксированы на протяжении всего жизненного цикла компонента. Для данных, которые мы собираемся менять – мы должны использовать state.

Обычно, следует инициализировать state в конструкторе, потом вызвать функцию setState когда хотите что-то поменять.

Для примера, сделаем текст, который будет всё время мелькать. Текст устанавливается один раз, когда создаётся мелькающий компонент, так что текст – это prop. Состояние «Включён сейчас текст или выключен» со временем меняется, так что это – state.

В настоящем приложении, вы, вероятнее всего, не будете ставить таймер на state. Скорее всего вы будете устанавливать state, когда получите новые данные с сервера или от пользователя. Также вы можете использовать контейнер состояний Redux или Mobx , чтобы контролировать поток данных. В таком случае, лучше использовать Redux или Mobx, чтобы изменять состояние, чем использовать функцию setState.

Когда вызван setState, BlinkApp перерендерит этот компонент. Вызывая setState по таймеру, компонент будет перерендериваться каждый щелчок таймера.

State работает так же, как и в React, поэтому, для большей информации по state, вы можете почитать React.Component API . После этого примера, вам может наскучить унылый чёрный текст в приложении. Чтобы делать более красивые вещи, вам надо прочитать о Style.
