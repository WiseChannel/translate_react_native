Layout with Flexbox (макеты Flexbox)

Компонент может определять расположение его детей, используя flexbox алгоритм.Flexbox разработан для обеспечения согласованного размера на разных экранах.

Вы, в основном, будете использовать комбинации из flexDirection , alignItems и justifyContent , чтобы достичь правильного расположения.

Flexbox работает так же в React Native, как и в CSS, но с некоторыми отличиями. Значения по умолчанию отличаются: во flexDirection используется column , а не row, а параметр flex поддерживает только одно число.

Flex Direction

Добавление flexDirection в style компонента определяет основную ось его компонента. Должны ли дети быть расположены горизонтально (row) или вертикально (column)? По умолчанию это column.

Justify Content

Добавление justifyContent к стилю компонента определяет распределение дочерних элементов вдоль основной оси. Должны ли дети распределятся в начале, центре, конце или располагаться равномерно? Доступные опции: flex-start (в начале), center (в центре), flex-end (в конце), space-around (вокруг), space-between(между) и space-evently(по порядку).

Align Items

Добавление alignItems в стиль компонента определяет выравнивание дочерних элементов вдоль вторичной оси (если первичной осью является строка, тогда вторичной – столбец, и наоборот). Должны ли дети быть выровнены в начале, в центре, в конце или растянуты до заполнения? Доступные опции: flex-start (в начале), center (в центре), flex-end (в конце) и stretch(до заполнения).

Чтобы растяжение имело эффект, дочерние элементы не должны иметь фиксированный размер вдоль вторичной оси. В следующем примере установка alignItems: stretch ничего не делает, пока width: 50 не будет удалена из дочерних элементов.
