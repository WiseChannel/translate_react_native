Props

Большинство компонентов можно изменять когда они уже созданы с разными параметрами. Эти параметры создания называются props.

Для примера, один из основных компонентов React Native – это Image. Когда вы делаете картинку, вы можете использовать prop, называемый source, чтобы контролировать какую картинку он покажет.

Обратите внимание на скобки, окружающие {pic} – они вставляют переменную pic в JSX. Вы можете поместить любое JavaScript выражение внутри скобок в JSX.

Ваш собственный компонент также может использовать props. Это позволяет вам сделать единственный компонент, который используется во многих местах вашего приложения, с немного разными параметрами в разных местах. Просто обратитесь к this.props в вашей функции рендеринга. Вот пример:

Этот пример также использует Greeting компонент в JSX, прямо как встроенные компоненты. Возможность сделать это – то, что делает React таким крутым,- если вы хотите, чтобы у вас был другой набор примитивов пользовательского интерфейса, вы просто изобретаете другие.

Другие нове вещи здесь – это View компонент. View полезный в качестве контейнера для других компонентов, чтобы помогать контролировать стиль и макет.

С props и основными Text , Image и View компонентами, вы можете делать много разных статичных экранов. Чтобы понять как сделать так, чтобы ваше приложение менялось со временем, вам надо прочитать о State.
